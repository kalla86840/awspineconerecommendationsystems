Pinecone Recommendation Systems Task Runbook
===========================================

Purpose
-------
Use Pinecone as the vector database for the AWS OpenAI recommendation systems
real-time endpoint deployed through CodePipeline. The primary task embeds a
seed item, seed text, or user interest profile with OpenAI using
1024-dimensional vectors, queries Pinecone, and returns the closest records as
ranked recommendations. Semantic search remains available as a compatibility
mode over the same 1024-dimensional index.

AWS Secrets Manager
-------------------
Create or update the Pinecone API key secret:

PowerShell:
.\infrastructure\create-pinecone-secret.ps1 -PineconeApiKey "pcsk-your-pinecone-api-key"

Bash:
./infrastructure/create-pinecone-secret.sh "pcsk-your-pinecone-api-key"

Expected secret ARN:
arn:aws:secretsmanager:us-west-1:659613508664:secret:awspineconeapikey1-kiudra

Environment Variables
---------------------
PINECONE_API_KEY_SECRET_ARN=arn:aws:secretsmanager:us-west-1:659613508664:secret:awspineconeapikey1-kiudra
PINECONE_INDEX_NAME=news-demo
PINECONE_INDEX_HOST=https://news-demo-4fe9eo0.svc.aped-4627-b74a.pinecone.io
PINECONE_NAMESPACE=news
PINECONE_CLOUD=aws
PINECONE_REGION=us-east-1
PINECONE_DIMENSION=1024
PINECONE_UPSERT_ON_QUERY=false
OPENAI_EMBEDDING_MODEL=text-embedding-3-small

Recommendation Systems Request
------------------------------
Use this request body to ask Pinecone for similar records. This is the primary
CodePipeline smoke test for this repository:

{
  "mode": "recommendations",
  "seed_text": "Recent earnings call transcripts and market news mentioning edge cloud, content delivery, and EGIO.",
  "interests": [
    "earnings calls",
    "edge cloud",
    "content delivery",
    "EGIO"
  ],
  "top_k": 5
}

You can also use a Pinecone record as the seed when you already know its id:

{
  "mode": "recommendations",
  "seed_id": "record-id-from-news-demo",
  "top_k": 5
}

Semantic Search Compatibility Request
-------------------------------------
Use this request body when you need the older semantic search behavior:

{
  "mode": "semantic_search",
  "question": "What recent earnings call transcripts mention EGIO?",
  "top_k": 5
}

Curl Test
---------
Replace ENDPOINT_URL with the CloudFormation output named EndpointUrl:

curl -X POST "$ENDPOINT_URL" ^
  -H "content-type: application/json" ^
  -d @samples/pinecone_recommendations_request.json

curl -X POST "$ENDPOINT_URL" ^
  -H "content-type: application/json" ^
  -d @samples/pinecone_semantic_search_request.json

Expected Response Fields
------------------------
For recommendation requests, recommendations contains ranked similar records,
including id, title, content, score, and metadata. When seed_id is used, the
seed record is excluded from the returned recommendations.

For semantic search compatibility requests, retrieved_context contains ranked
Pinecone matches, including id, title, content, score, and the original
Pinecone metadata.

CodePipeline Behavior
---------------------
The endpoint deployment buildspec runs the Pinecone recommendation systems
smoke test first after CloudFormation deploys the Lambda Function URL. It then
runs semantic search as a compatibility check. Both use 1024-dimensional OpenAI
embeddings and must query a Pinecone index with dimension 1024.

Troubleshooting
---------------
If retrieval_source is "keyword", confirm PINECONE_API_KEY_SECRET_ARN is set and
the Lambda role can read the Pinecone secret.

If Pinecone returns no records, confirm the namespace is news and that records
exist in news-demo. Leave PINECONE_UPSERT_ON_QUERY=false when querying the
existing news index so the endpoint does not insert local sample documents.

If dimensions mismatch, confirm the Pinecone index dimension matches
PINECONE_DIMENSION and the OpenAI embedding model dimension. The current project
uses text-embedding-3-small with 1024 dimensions.
